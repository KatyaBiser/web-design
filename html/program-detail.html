Semantic HTML Recovery

4 weeks • Beginner to Intermediate • Led by Dr. Emma HTML

Program Overview

Break free from div-soup and embrace the power of semantic markup. This comprehensive 4-week program will teach you to structure content with meaning, improve SEO, and make your websites accessible to all users.

Through a combination of lectures, hands-on exercises, and personalized code reviews, you'll learn to write HTML that is not just functional, but meaningful and accessible. Say goodbye to nested divs and hello to proper document structure.

What You'll Learn

  • Understanding HTML5 semantic elements and their proper use
  • Creating accessible document structures with headings hierarchy
  • Using <article>, <section>, <aside>, and <nav> effectively
  • Implementing ARIA roles and attributes when necessary
  • Building accessible forms with proper labels and validation
  • Understanding the <time> element and microdata
  • Creating semantic tables with <thead>, <tbody>, and <caption>
  • Testing your markup with screen readers and validation tools
  • Refactoring existing div-heavy code into semantic HTML

Prerequisites

This program is designed for developers who:

  • Have basic HTML knowledge but want to improve
  • Rely heavily on <div> elements for all structure
  • Have never learned proper semantic markup
  • Want to improve accessibility in their projects
  • Are ready to commit 6-8 hours per week to learning

Program Format

Each week includes:

  • 2 live sessions (90 minutes each) - Tuesday and Thursday evenings
  • Self-paced exercises - Approximately 3 hours per week
  • Code review session - 30 minutes individual feedback
  • Online community access - Ask questions anytime

Expected Outcomes

By the end of this program, you will be able to:

  • Write semantic HTML confidently without consulting documentation
  • Create accessible websites that meet WCAG 2.0 AA standards
  • Understand the impact of markup choices on SEO and accessibility
  • Use screen readers to test your own work
  • Refactor existing code to be more semantic and maintainable
  • Make informed decisions about when to use different HTML5 elements
