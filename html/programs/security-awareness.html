Security Awareness

4 weeks • Intermediate to Advanced • Led by Dr. Sam Security

Program Overview

Your API keys are in your GitHub repo. Your database password is "password123". You're concatenating user input directly into SQL queries. Your authentication system is "just check if the username matches." You store passwords in plain text because "encryption is complicated." You don't sanitize inputs because "who would even try to hack my small website?"

Then one day, someone adds `'; DROP TABLE users; --` to your contact form. Your database is gone. Your user data is on Pastebin. Your AWS bill is $47,000 because someone found your keys and mined Bitcoin. You're frantically googling "how to undo SQL injection" at 3 AM. This could have been prevented. Security isn't optional—it's mandatory.

This intensive 4-week program, led by Dr. Sam Security (who has stopped more breaches than you've had hot dinners), will teach you that security is not something you "add later." It's baked into every line of code you write. Because when you get hacked, "I didn't know" is not a valid excuse.

What You'll Learn

  • OWASP Top 10 vulnerabilities (memorize them, live them, prevent them)
  • SQL injection prevention (parameterized queries are your friends)
  • XSS (Cross-Site Scripting) attacks and how to stop them
  • CSRF (Cross-Site Request Forgery) protection
  • Password hashing (bcrypt, not MD5, dear god not MD5)
  • Authentication vs Authorization (they're not the same thing!)
  • Input validation and sanitization (trust NO input, ever)
  • API security: keys, tokens, and why they shouldn't be in git
  • HTTPS, SSL/TLS, and why HTTP is dead to us
  • Security headers (CSP, HSTS, X-Frame-Options)

Prerequisites

This program is for developers who:

  • Have API keys committed to GitHub (and now you're panicking)
  • Store passwords in plain text or use MD5 hashing
  • Concatenate user input into SQL queries (oh no)
  • Think "my site is too small to be hacked" (narrator: it wasn't)
  • Don't know what XSS, CSRF, or SQL injection actually are
  • Have never heard of OWASP
  • Validate inputs on the frontend only (hilarious)
  • Are ready to commit 12-15 hours per week to not getting hacked

Program Format

Each week includes:

  • 4 intensive sessions (2 hours each) - Mon/Tue/Thu/Fri evenings
  • Capture The Flag challenges - Hack vulnerable apps, then fix them
  • Security audits - Review real code for vulnerabilities
  • Penetration testing practice - Break things legally
  • Code review sessions - Find security flaws in peer code
  • Emergency response drills - What to do when you get breached

Expected Outcomes

By the end of this program, you will:

  • Understand and prevent OWASP Top 10 vulnerabilities
  • Write SQL queries that can't be injected
  • Prevent XSS attacks with proper output encoding
  • Implement CSRF protection in your apps
  • Hash passwords properly (bcrypt, argon2, not MD5!)
  • Validate and sanitize ALL user inputs
  • Never commit secrets to version control again
  • Implement security headers correctly
  • Conduct basic security audits on your own code
  • Sleep better knowing your users' data is protected
