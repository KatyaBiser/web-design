Bootstrap Detox

8 weeks • All Levels • Led by Specialist Val Root

Program Overview

Let's be honest: you've been using Bootstrap (or Tailwind, or Material-UI, or whatever framework is trending this week) because you don't actually know CSS. You just memorized that `class="container"` does... something? And `row` and `col-md-6` magically make columns, probably? You've been copy-pasting classes from documentation like a CSS-illiterate raccoon raiding a trash can.

"But I'm productive with frameworks!" you cry. Sure, Jan. Your 500KB CSS bundle for a landing page says otherwise. Your inability to center a div without consulting Bootstrap docs says otherwise. The fact that you're loading jQuery in 2025 "because Bootstrap needs it" says otherwise.

This 8-week intensive program will teach you to write custom, lightweight CSS that actually makes sense. Led by Specialist Val Root, who has successfully detoxed over 300 developers from framework dependency. Val once built a production app with zero frameworks—just vanilla CSS, semantic HTML, and audacity.

What You'll Learn

  • Understanding the CSS box model (yes, for real this time)
  • Writing custom grid systems without importing 47 utility classes
  • Creating responsive layouts using actual media queries you understand
  • Building component libraries without frameworks (gasp!)
  • CSS custom properties (variables) - no preprocessor required
  • Understanding specificity so you stop adding `!important` everywhere
  • Naming conventions that make sense (BEM, SMACSS, or roll your own)
  • CSS architecture: organizing styles without a framework's help
  • Performance optimization: loading 5KB instead of 500KB
  • Accepting that you don't need a framework for every project

Prerequisites

This program is for developers who:

  • Have Bootstrap or Tailwind in every project's dependencies
  • Can't build a navbar without copy-pasting from Bootstrap docs
  • Have never written a media query from scratch
  • Think "col-md-6" is actual CSS knowledge
  • Spend more time fighting framework defaults than writing custom styles
  • Have 90% unused CSS in their production bundles
  • Are ready to commit 10-12 hours per week to CSS enlightenment
  • Want to understand what their CSS actually does
