### Требования к веб-части (HTML/CSS)

Вам нужно создать веб-сайт на основе ранее сделанного прототипа.

**Структура и содержание:**
*   **Два варианта на выбор:**
    1.  Создать **главную страницу + еще одну** страницу с использованием HTML и CSS, а также сделать **кликабельный прототип (High-fidelity wireframe) всех страниц** сайта.
    2.  Создать **главную страницу + еще одну** страницу с использованием HTML и CSS, а также сделать **кликабельный прототип всех страниц** сайта.
*   Сайт должен иметь **логический смысл**.
*   Все тексты, изображения и иллюстрации не должны быть сгенерированы искусственным интеллектом.
*   Должно быть **четко указано**, что это студенческий проект (идеально — в футере/подвале каждой страницы).
*   **Минимальное количество обязательных страниц:**
    *   Полноценная **главная страница** (Homepage).
    *   **Список** обследований/отделений/врачей.
    *   **Детальная страница** обследования/отделения/врача.
    *   **Результаты поиска** (с поисковой строкой на всех страницах). Поиск может не быть функциональным, достаточно показать, как выглядят результаты по определенному запросу.
    *   **Контакты** (как вас найти и т.д.).
    *   Возможность **резервации** визита/обследования (полный процесс).
*   **Полноценный (расширенный) футер**, в котором должны быть указаны имена авторов, а не просто "copyright" и имена.

**Технические требования к HTML/CSS:**
*   **Основа:** Код должен базироваться на созданном прототипе (та же структура, тот же стиль, но улучшенный внешний вид).
*   **Адаптивный дизайн (Responsive):** Сайт должен корректно отображаться на разных устройствах.
*   **Подход "Mobile-first":** Разработку нужно начинать с мобильной версии.
*   **HTML5:** Использовать последнюю версию (не XHTML, HTML 4).
*   **Семантическая верстка:** Правильное использование семантических элементов.
*   **Обязательные элементы к использованию:**
    *   `article`, `section`, `header`, `footer`, `aside`, `time`, `table`
    *   `p`, `img`
    *   `h1`, `h2`, `h3`
*   **CSS:**
    *   **Без фреймворков и шаблонов.**
    *   Использовать **1 собственный файл CSS** размером не менее 400 строк.
    *   Использовать **CSS переменные** (custom properties) для цветов, отступов, размеров шрифта.
    *   **Grid Layout:** Использовать для основной сетки. Должно быть минимум 8 элементов в гриде, из которых как минимум 2 должны занимать несколько ячеек (grid cells). Весь элемент не должен легко решаться с помощью flexbox.
    *   **Flexbox.**
*   **Доступность (Accessibility):** Сайт должен соответствовать стандарту WCAG 2.0 уровня AA или выше/новее.
*   **Относительные пути:** Всегда использовать относительные адреса для ссылок и ресурсов.

### Требования к кликабельному прототипу (Wireframe)

*   **Полноценный/комплектный веб:** Прототип должен представлять собой полную версию сайта.
*   **Адаптивность:** Должны быть версии для мобильного телефона и для десктопа.
*   **Интерактивность:** Все ссылки должны быть кликабельными. Например, ссылки из списка должны вести на одну и ту же детальную страницу.
*   **Формат:** Прототип должен запускаться в любом веб-браузере без необходимости установки дополнительных приложений.
*   **Рекомендуемый инструмент:** Axure RP.
*   **Сдача:** Экспорт в формате HTML или PDF. Если экспорт не будет полностью кликабельным, можно предоставить ссылку на рабочую онлайн-версию.
